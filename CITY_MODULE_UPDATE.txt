FUNFERNUS BOT — МОДУЛЬ «РЕЕСТР, МОДЕРАЦИЯ И УПРАВЛЕНИЕ ГОРОДАМИ»

Модуль находится в modules/cities.py и подключён как дополнение. Системы бизнеса,
банка, основных заявок, Community/поддержки, Government, тикетов и RCON не переписаны.

ОБЯЗАТЕЛЬНАЯ НАСТРОЙКА
1. Создайте пять каналов:
   - текстовый канал подачи заявок;
   - закрытый текстовый канал рассмотрения;
   - публичный Discord Forum для реестра;
   - закрытый текстовый канал управления городом;
   - закрытый текстовый канал логов городов.
2. Создайте обычную роль «Мэр» и поставьте роль бота выше неё.
3. В канале управления разрешите просмотр роли «Мэр» и администрации.
4. Выдайте боту права:
   - View Channel, Send Messages, Embed Links, Attach Files;
   - Read Message History, Manage Messages;
   - Create Public Threads, Send Messages in Threads, Manage Threads;
   - Manage Roles для выдачи роли «Мэр».
5. Запустите /настроить_бота → «Города» либо /настроить_города.
6. В панели выберите пять каналов, роль мэра, роли администраторов/модераторов
   и при необходимости разрешённых сторонних ботов.
7. Нажмите «Опубликовать панели».

ИЗОБРАЖЕНИЯ И БАННЕРЫ
- Все баннеры системы городов лежат локально в assets/banners/cities/.
- Пользовательские материалы сохраняются в assets/city_uploads/<guild_id>/<CITY-ID>/.
- Embed получает изображение только через attachment://имя_файла и настоящий discord.File.
- Внешние URL изображений нигде не выводятся и не используются для баннеров.
- Скриншоты города отправляются отдельным сообщением как настоящие вложения.
- Основная карточка и сообщение со скриншотами имеют отдельные ID в JSON.
- На хостинге папка проекта должна быть постоянной. Если хостинг очищает файловую
  систему при каждом перезапуске, подключите постоянный диск/volume либо не выполняйте
  полный сброс папки assets/city_uploads.

ПРАВА В ПУБЛИКАЦИЯХ ФОРУМА
- Писать могут только текущий мэр, текущий заместитель, настроенная администрация,
  основной бот и выбранные в настройке разрешённые боты.
- Проверка выполняется по Discord ID. Ник и отображаемое имя на права не влияют.
- Постороннее сообщение удаляется, попытка журналируется, а автор получает ЛС.
- Если ЛС закрыты, предупреждение появляется в публикации и удаляется через несколько секунд.
- Для предупреждений действует ограничение частоты, чтобы не создавать спам.

УПРАВЛЕНИЕ РУКОВОДСТВОМ
- Команда: /город_руководство CITY-0001
- Команда доступна только настроенным администраторам и модераторам.
- После замены руководителя список разрешённых Discord ID обновляется сразу.
- Старые сообщения бывшего руководителя не удаляются.
- Изменение записывается в логи и отдельным служебным Embed в публикацию города.

ОТКАЗОУСТОЙЧИВОСТЬ
- После запуска бот проверяет руководителей, публикации, основные сообщения,
  сообщения со скриншотами и локальные файлы.
- Удалённая публикация или сообщение фиксируется в JSON и логах.
- Уход мэра/заместителя с сервера не вызывает падение; в карточке появляется статус,
  а администрация получает запись о необходимости замены.
- Требуется discord.py==2.7.1: FileUpload внутри Modal появился в этой версии.
